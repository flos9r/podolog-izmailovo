# Сайт подолога — Наталья Сунцова, Измайлово

Статический лендинг для подолога. Чистый HTML/CSS/JavaScript, без зависимостей и сборщиков.

---

## 🔍 Где находятся изменения и как их проверить

Все изменения сайта находятся в **Pull Request (PR)** — это предварительная проверка перед публикацией.

### Шаг 1. Откройте Pull Request

Перейдите по ссылке:
👉 **https://github.com/flos9r/podolog-izmailovo/pull/1**

Или: откройте репозиторий → вкладка **Pull requests** → выберите PR #1.

### Шаг 2. Посмотрите изменения

На странице PR:
- Вкладка **Files changed** — показывает все изменённые файлы (зелёное = добавлено, красное = удалено).
- Вкладка **Commits** — показывает список коммитов (отдельных изменений).

### Шаг 3. Проверьте сайт локально (по желанию)

```bash
# 1. Скачайте код на компьютер
git clone https://github.com/flos9r/podolog-izmailovo.git
cd podolog-izmailovo

# 2. Сайт уже на правильной ветке — просто откройте index.html
#    Двойной клик по index.html или:
python3 -m http.server 8080
# Затем откройте http://localhost:8080 в браузере
```

### Шаг 4. Примените изменения (Merge)

Когда всё устраивает:
1. На странице PR нажмите зелёную кнопку **«Ready for review»** (если PR в статусе Draft).
2. Затем нажмите **«Merge pull request»** → **«Confirm merge»**.
3. Изменения появятся в основной ветке `main`.

### Шаг 5. Включите GitHub Pages (публикация сайта)

1. Перейдите в **Settings** репозитория (⚙️ иконка шестерёнки).
2. В боковом меню выберите **Pages**.
3. В разделе **Source** выберите: **Deploy from a branch**.
4. Ветка: **main**, папка: **/ (root)**.
5. Нажмите **Save**.
6. Через 1–2 минуты сайт будет доступен по адресу:
   👉 **https://flos9r.github.io/podolog-izmailovo/**

## Как открыть проект локально

1. Скачайте или склонируйте репозиторий на компьютер.
2. Откройте папку проекта в любом редакторе (VS Code, WebStorm, Sublime Text).
3. Чтобы открыть сайт в браузере, воспользуйтесь любым из способов:
   - Установите расширение **Live Server** (VS Code) и нажмите "Go Live".
   - Или запустите локальный сервер командой: `python3 -m http.server 8080` — затем откройте `http://localhost:8080`.
   - Или просто откройте файл `index.html` в браузере двойным кликом (большинство функций работает).

## Как добавить фотографии

### Фото специалиста (hero-блок)

1. Положите файл в папку `images/` с именем `specialist.jpg`.
2. Откройте `index.html` и найдите (Ctrl+F): `hero__photo-placeholder`.
3. Замените `<div class="hero__photo-placeholder" ...>...</div>` на:
   ```html
   <img class="hero__photo" src="images/specialist.jpg"
        alt="Наталья Сунцова — подолог в Измайлово"
        width="400" height="500" />
   ```

### Фото в разделе "Специалист"

1. Положите файл в `images/about.jpg`.
2. В `index.html` найдите: `about__photo-placeholder`.
3. Замените `<div class="about__photo-placeholder" ...>...</div>` на:
   ```html
   <img class="about__photo" src="images/about.jpg"
        alt="Наталья Сунцова — подолог в рабочем кабинете"
        width="600" height="750" />
   ```

### Фотографии работ (галерея)

1. Положите файлы в `images/gallery/work-1.jpg`, `work-2.jpg`, ..., `work-6.jpg`.
2. Откройте `js/data.js` и найдите массив `gallery:`.
3. Для каждого изображения укажите правильный путь в поле `src`.

Все изображения на сайте защищены от выхода за контейнер через `max-width: 100%` и `object-fit: cover`.
Для зарезервированного места под изображение используйте CSS-класс `.img-wrap` с модификатором:
- `.img-wrap--portrait` — соотношение 4:5 (портрет)
- `.img-wrap--banner` — соотношение 16:9 (горизонтальный баннер)
- `.img-wrap--square` — квадрат 1:1

## Как изменить доступные записи (расписание)

Откройте файл `js/data.js` и найдите массив `availableSlots:`.

Каждый элемент массива — это один день:

```javascript
availableSlots: [
  {
    date:  "2026-04-01",        // дата в формате YYYY-MM-DD
    label: "Вт, 1 апреля",     // как будет показано пользователю
    times: ["10:00", "12:00", "16:00"],  // доступные слоты этого дня
  },
  // добавьте следующий день ниже...
]
```

Что нужно знать:
- Прошедшие даты скрываются автоматически — удалять их необязательно.
- Выбранное время автоматически подставляется в форму записи.
- После изменения файла достаточно обновить страницу браузера.

## Как проверить мобильную версию

Способ 1 — инструменты разработчика браузера:
1. Откройте сайт в Chrome или Firefox.
2. Нажмите F12 → включите режим устройства (иконка телефона или Ctrl+Shift+M).
3. Выберите разрешение: 320px, 360px, 390px или 412px.
4. Прокрутите страницу — убедитесь, что нет горизонтального скролла.

Способ 2 — реальное устройство:
1. Убедитесь, что телефон и компьютер в одной Wi-Fi сети.
2. Запустите `python3 -m http.server 8080`.
3. Узнайте IP компьютера (`ipconfig` / `ip a`) и откройте `http://192.168.x.x:8080` на телефоне.

## Как изменить контент (тексты, цены, контакты)

Весь контент хранится в `js/data.js`. Редактируйте объект `siteData`:

- `specialist` — имя, должность, описание, сертификаты
- `services` — список услуг с описаниями
- `prices` — прайс-лист
- `gallery` — пути к фото работ
- `reviews` — отзывы (имя, дата, рейтинг, текст)
- `advantages` — блок "Почему наш кабинет"
- `contacts` — телефон, email, адрес, мессенджеры
- `availableSlots` — доступные даты и время для записи

## Личный кабинет подолога (Base44)

На сайте добавлена ссылка на внешний личный кабинет подолога, размещённый на платформе **Base44**.
Кабинет предназначен **только для специалиста** — клиенты записываются через форму на сайте.

### Что это даёт

- Управление записями и расписанием через внешний сервис.
- Ссылка открывается в новой вкладке, безопасно (`target="_blank"`, `rel="noopener noreferrer"`).
- Кабинет **не встроен** в сайт через iframe — используется обычная внешняя ссылка.

### Где отображается ссылка

| Место         | Элемент                                       |
|---------------|-----------------------------------------------|
| Header        | Ссылка «Для специалиста» рядом с кнопкой «Записаться» |
| Footer        | Ссылка «Вход для специалиста» в нижней части  |
| Мобильное меню | Ссылка «Вход для специалиста» под кнопкой «Записаться» |

### Где менять URL кабинета

URL хранится в **одном месте**: `js/data.js` → поле `siteData.cabinetUrl`.

```javascript
const siteData = {
  cabinetUrl: "https://podolog-pro-flow.base44.app/bookings",
  // ...
};
```

При изменении этого значения ссылка обновится автоматически в header, footer и мобильном меню.

### Важно: разделение ролей

- **Форма записи** (раздел «Записаться на приём») — для **клиентов**.
- **Личный кабинет** (Base44) — для **подолога** (управление записями, расписание).
- Это **разные части системы**. Форма записи отправляет данные через Formspree на почту специалиста. Личный кабинет — внешний сервис для управления.

## Перенос на другой домен / хостинг

Подробная инструкция: **[docs/migration.md](docs/migration.md)**

Краткий чек-лист:

1. Склонируйте репозиторий
2. Скопируйте `.env.example` → `.env` и заполните новые значения
3. Обновите контакты в `js/data.js` → объект `contacts`
4. Обновите Formspree endpoint в `js/form.js` (если нужно)
5. Обновите SEO-мета в `index.html` (строки 6–11)
6. Обновите адрес и ссылку на карту в `index.html`
7. Задеплойте на хостинг (GitHub Pages / Netlify / VPS / Docker)

Сайт не требует сборки, backend-а или базы данных — его можно разместить на любом статическом хостинге.
