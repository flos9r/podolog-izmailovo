# Полный гайд по добавлению фотографий

Этот файл — единственный документ, который нужно прочитать, чтобы правильно добавить фото на сайт.

## 1. Общие правила для всех фото

Перед тем как добавлять фото, прочитайте эти три правила — они касаются всех изображений.

### Формат файла
Используйте **JPEG** (расширение `.jpg`).  
PNG тоже работает, но JPEG быстрее загружается и достаточен для фото.

### Размер файла
Максимум **500 КБ** на одно фото. Оптимально — **150–300 КБ**.  
Файлы больше 1 МБ заметно замедляют загрузку страницы на телефоне.

**Как уменьшить размер:**
- Зайдите на [squoosh.app](https://squoosh.app) (бесплатно, работает в браузере).
- Загрузите фото → выберите **MozJPEG** → ползунок Quality на 75–80 → нажмите **Сохранить**.

### Имена файлов
- Используйте только **латинские буквы, цифры и дефисы**.  
  ✅ `work-7.jpg`, `specialist.jpg`, `about.jpg`  
  ❌ `Фото Натальи.jpg`, `work 7.jpg`, `фото.jpg`
- Строчные буквы (без заглавных).

## 2. Фото на первом экране (hero)

Это главное фото специалиста — оно стоит на самом видном месте, в левой колонке первого экрана.

### Что нужно сфотографировать
- Портрет или поясной снимок специалиста.
- Нейтральный или светлый фон.
- Хорошее освещение, лицо хорошо видно.
- Лучше вертикальная ориентация (книжная).

### Рекомендуемый размер
- **600 × 750 px** или больше (пропорция примерно 4:5).
- Не меньше 400 px по ширине.

### Шаг 1 — Положите файл в папку

Переименуйте файл в `specialist.jpg` и положите его в папку `images/`:

```
images/
└── specialist.jpg   ← сюда
```

### Шаг 2 — Откройте index.html

Откройте файл `index.html` в редакторе (VS Code).  
Нажмите **Ctrl+F** (или Cmd+F на Mac) → введите `hero__photo-placeholder` → нажмите Enter.

Вы увидите такой блок (примерно строка 97):

```html
<div class="hero__photo-wrap image-shell hero-image-shell">
  <div class="hero__photo-placeholder" aria-hidden="true">
    <span>Фото специалиста</span>
  </div>
</div>
```

### Шаг 3 — Замените placeholder на img

Удалите только строки с `hero__photo-placeholder`:

```html
<div class="hero__photo-placeholder" aria-hidden="true">
  <span>Фото специалиста</span>
</div>
```

И вставьте на их место:

```html
<img
  class="hero__photo image-shell__img"
  id="specialist-photo"
  src="images/specialist.jpg"
  alt="Наталья Сунцова — подолог в Измайлово"
/>
```

Итоговый блок должен выглядеть так:

```html
<div class="hero__photo-wrap image-shell hero-image-shell">
  <img
    class="hero__photo image-shell__img"
    id="specialist-photo"
    src="images/specialist.jpg"
    alt="Наталья Сунцова — подолог в Измайлово"
  />
</div>
```

### Шаг 4 — Сохраните файл

Нажмите **Ctrl+S** → откройте сайт в браузере и проверьте.

## 3. Фото в разделе «Специалист»

Это фото стоит в разделе «О специалисте» — рядом с текстом биографии.

### Что нужно сфотографировать
- Специалист в кабинете, за работой или в профессиональной обстановке.
- Горизонтальная или квадратная ориентация подходит лучше.
- Желательно, чтобы было видно кабинет или инструменты.

### Рекомендуемый размер
- **700 × 700 px** или больше (квадрат или чуть вытянутый по высоте).

### Шаг 1 — Положите файл в папку

Переименуйте файл в `about.jpg` и положите его в папку `images/`:

```
images/
├── specialist.jpg
└── about.jpg        ← сюда
```

### Шаг 2 — Откройте index.html

Откройте `index.html` → нажмите **Ctrl+F** → введите `about__photo-placeholder` → нажмите Enter.

Вы увидите такой блок (примерно строка 165):

```html
<div class="about__photo-placeholder" aria-hidden="true">
  <span>Фото будет добавлено</span>
</div>
```

### Шаг 3 — Замените placeholder на img

Удалите эти строки:

```html
<div class="about__photo-placeholder" aria-hidden="true">
  <span>Фото будет добавлено</span>
</div>
```

И вставьте на их место:

```html
<img
  class="about__photo"
  src="images/about.jpg"
  alt="Наталья Сунцова — подолог в рабочем кабинете"
/>
```

### Шаг 4 — Сохраните файл

Нажмите **Ctrl+S** → проверьте в браузере.

## 4. Галерея работ

Это сетка фотографий в разделе «Галерея» — снимки результатов работы до/после.

### Что нужно сфотографировать
- Результаты процедур: до и после, крупный план ногтей или стоп.
- Горизонтальная или квадратная ориентация.
- Хорошее освещение, чёткое фото.

### Рекомендуемый размер
- **600 × 450 px** или больше.
- Соотношение сторон 4:3 или 1:1.

### Текущие записи в галерее

Сейчас в данных прописаны 6 позиций (work-1.jpg … work-6.jpg).  
Если файлов нет — на их месте будет заглушка с подписью. Это нормально.

### Шаг 1 — Положите файлы в папку

Переименуйте фото в формате `work-N.jpg` (где N — номер) и положите в `images/gallery/`:

```
images/
└── gallery/
    ├── work-1.jpg    ← уже есть в данных
    ├── work-2.jpg
    ├── ...
    ├── work-6.jpg
    └── work-7.jpg    ← новое фото
```

Нумерация может быть любой, главное — совпадение с именем в данных (шаг 2).

### Шаг 2 — Добавьте запись в data.js

Откройте `js/data.js` → нажмите **Ctrl+F** → введите `gallery: [` → нажмите Enter.

Вы увидите массив из объектов. Пример одного объекта:

```js
{
  src: "images/gallery/work-1.jpg",
  alt: "Результат медицинского педикюра — до и после",
  category: "Педикюр",
},
```

Чтобы добавить новое фото — вставьте новый объект перед закрывающей `]`:

```js
gallery: [
  {
    src: "images/gallery/work-1.jpg",
    alt: "Результат медицинского педикюра — до и после",
    category: "Педикюр",
  },
  // ... остальные фото ...
  {
    src: "images/gallery/work-7.jpg",       // ← новое
    alt: "Обработка вросшего ногтя — до и после",
    category: "Вросший ноготь",
  },
],
```

### Поля объекта

| Поле       | Обязательное | Что писать                                                                  |
|------------|:------------:|-----------------------------------------------------------------------------|
| `src`      | ✅           | Путь к файлу, всегда начинается с `images/gallery/`                        |
| `alt`      | ✅           | Краткое описание фото (важно для доступности и SEO)                        |
| `category` | ✅           | Категория фильтра. Варианты ниже ↓                                          |

### Доступные категории

Категория появится в фильтре галереи автоматически.  
Используйте эти значения (или создайте новые — они добавятся в фильтр сами):

| Значение         | Когда использовать                              |
|------------------|-------------------------------------------------|
| `Педикюр`        | Медицинский педикюр, обработка стопы            |
| `Вросший ноготь` | Коррекция, тампонирование, скобы                |
| `Мозоли`         | Мозоли, натоптыши                               |
| `Коррекция`      | Коррекция формы ногтя                           |
| `Обработка`      | Аппаратная обработка, общая обработка стоп      |

### Шаг 3 — Удалите запись, если фото ещё нет

Если в данных есть запись, а файла — нет, на сайте появится заглушка.  
Это не ломает сайт, но выглядит некрасиво. Либо добавьте файл, либо удалите объект из `gallery: [`.

### Шаг 4 — Сохраните data.js

Нажмите **Ctrl+S** → перезагрузите страницу в браузере.

## 5. Как проверить результат

1. **Откройте сайт в браузере.**  
   Если используете VS Code + Live Server — он обновится автоматически.  
   Если нет — перезагрузите вкладку (F5 или Ctrl+R).

2. **Проверьте на мобильном размере.**  
   Нажмите F12 → иконка телефона (Toggle Device Toolbar) → выберите ширину 375px.

3. **Если фото не появилось:**
   - Убедитесь, что файл лежит в нужной папке.
   - Проверьте имя файла (латиница, без пробелов, строчные).
   - Убедитесь, что в `src` указан правильный путь (начинается с `images/`, не с `/images/`).
   - Откройте DevTools (F12) → вкладка **Console** — там будут ошибки, если путь неверный.

4. **Если фото обрезается или выглядит неправильно** — это нормально, CSS подгоняет изображение под блок.  
   Если критично — выберите другое кадрирование при съёмке.

## 6. Частые ошибки

| Ошибка | Как исправить |
|--------|---------------|
| Файл есть, но фото не отображается | Проверьте путь в `src` — он должен совпадать с именем и расположением файла |
| В пути написано `/images/...` | Уберите ведущий слеш — правильно: `images/specialist.jpg` |
| Имя файла с кириллицей или пробелами | Переименуйте: только латиница, цифры, дефис |
| Фото появилось, но очень тяжёлое | Сожмите через [squoosh.app](https://squoosh.app) до 150–300 КБ |
| Галерея не обновилась | Убедитесь, что сохранили `data.js` и перезагрузили браузер |
| Placeholder не убирается | Убедитесь, что полностью удалили div с `photo-placeholder` и заменили его на `<img>` |
| Фото в галерее обрезается сверху/снизу | Сделайте снимок с более квадратной композицией (объект по центру) |

После добавления всех фото папка должна выглядеть так:

```
images/
├── specialist.jpg      ← фото на первом экране
├── about.jpg           ← фото в разделе «Специалист»
└── gallery/
    ├── work-1.jpg
    ├── work-2.jpg
    ├── work-3.jpg
    ├── work-4.jpg
    ├── work-5.jpg
    ├── work-6.jpg
    └── work-7.jpg      ← и так далее
```
